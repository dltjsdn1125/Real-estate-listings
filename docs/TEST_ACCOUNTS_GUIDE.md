# 테스트 계정 생성 가이드

## 🎯 목적
사용자 관리 페이지에서 승인 기능을 테스트하기 위한 테스트 계정 생성 방법

## 📋 생성할 테스트 계정 목록

### 1️⃣ 승인 대기 - 일반 회원
- **이메일**: `user1@test.com`
- **비밀번호**: `Test1234!@#`
- **이름**: `김철수`
- **전화번호**: `010-1111-2222`
- **회사명**: `철수상회`

### 2️⃣ 승인 대기 - 중개사
- **이메일**: `agent1@test.com`
- **비밀번호**: `Test1234!@#`
- **이름**: `박중개`
- **전화번호**: `010-3333-4444`
- **회사명**: `박중개사무소`

### 3️⃣ 승인 대기 - 프리미엄 회원
- **이메일**: `premium1@test.com`
- **비밀번호**: `Test1234!@#`
- **이름**: `강프리`
- **전화번호**: `010-6666-7777`
- **회사명**: `프리미엄부동산`

## 🚀 생성 방법

### 방법 1: 회원가입 페이지 이용 (추천)

1. 브라우저에서 회원가입 페이지 접속:
   ```
   http://localhost:3000/auth/signup
   ```

2. 위 테스트 계정 정보를 입력하여 회원가입

3. 각 계정은 자동으로 `pending` (승인 대기) 상태로 생성됨

4. 관리자 계정으로 로그인:
   - 이메일: `admin@daegu.com`
   - 비밀번호: `Admin1234!@#`

5. 사용자 관리 페이지 접속:
   ```
   http://localhost:3000/admin/users
   ```

6. "권한 승인 대기" 탭에서 승인 대기 중인 사용자 확인

7. 각 사용자의 "승인" 버튼 클릭하여 승인 처리

### 방법 2: SQL로 이메일 확인 처리 (회원가입 후)

회원가입 후 이메일 확인을 건너뛰려면:

```sql
-- 특정 사용자의 이메일 확인 처리
UPDATE auth.users 
SET email_confirmed_at = NOW()
WHERE email = 'user1@test.com';
```

## ✅ 테스트 시나리오

### 시나리오 1: 승인 프로세스 테스트
1. 테스트 계정 3개 생성 (위 계정 1, 2, 3)
2. 관리자 로그인
3. "권한 승인 대기" 탭에서 대기 중인 사용자 3명 확인
4. 각각 승인 버튼 클릭
5. "전체 사용자" 탭에서 승인된 사용자로 상태 변경 확인

### 시나리오 2: 필터링 테스트
1. 여러 등급의 사용자 생성 (bronze, silver, gold, premium)
2. 등급별 필터 선택하여 사용자 목록 필터링 확인
3. 상태별 필터 (approved, pending, rejected) 테스트

### 시나리오 3: 검색 테스트
1. 검색창에 이름, 이메일, 전화번호 입력
2. 실시간 검색 결과 확인

## 🔍 현재 관리 기능

### ✅ 구현 완료
- 사용자 목록 표시 (페이지네이션 포함)
- 등급(Tier) 및 상태(Status) 필터링
- 검색 기능 (이름, 이메일, 전화번호)
- 승인 대기 사용자 승인 버튼
- 사용자 통계 (전체, 대기, 프리미엄, 관리자)
- 반응형 디자인 (모바일 대응)
- 다크모드 지원

### 🎨 UI 특징
- 체크박스를 통한 일괄 선택
- 사용자 프로필 이미지 표시
- 등급별 색상 배지
- 상태별 색상 인디케이터
- 승인 대기 사용자 알림 배지

## 📊 통계 카드 설명

1. **전체 사용자**: 모든 사용자 수
2. **승인 대기**: pending 상태 사용자 수
3. **프리미엄 회원**: premium, gold 등급 사용자 수
4. **관리자**: admin 권한 사용자 수

## 💡 팁

- 테스트 계정은 모두 동일한 비밀번호(`Test1234!@#`)를 사용하여 관리 편의성 향상
- 각 계정은 실제 로그인 가능하므로 권한별 기능 테스트도 가능
- 테스트 후 불필요한 계정은 Supabase 대시보드에서 삭제 가능
